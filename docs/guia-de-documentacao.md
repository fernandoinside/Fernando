# Guia de Documentação dos Projetos

Use este modelo ao adicionar um novo produto na pasta `_projects/`.

## Campos principais

- `title`: nome do produto.
- `subtitle`: frase curta de apresentação.
- `description`: resumo para listagem e SEO.
- `status`: `disponivel`, `vendido`, `em_manutencao` ou `sob_demanda`.
- `category`: categoria do produto.
- `tech_stack`: tecnologias usadas.
- `pricing`: opções de licenciamento e valores.
- `docs_url`: link público da documentação, se existir.
- `demo_url`: link da demonstração, se existir.
- `repo_url`: link do repositório privado, se você quiser exibir.

## Conteúdo recomendado

1. Visão geral.
2. Funcionalidades.
3. Tecnologias.
4. Instalação e execução.
5. Estrutura do projeto.
6. Telas ou fluxos principais.
7. Documentação técnica.
8. Licença e condições comerciais.
9. Próximos passos e melhorias.
